## Bug Fixed During Build

A stray `renderer;` bare expression statement was accidentally left in `renderer.js` between two JSDoc comment blocks. In ES module strict mode this caused a silent reference error that killed the entire module graph. Fixed by removing the orphaned statement.

## Key Architecture Decisions

- **Dirty-flag render loop** — `renderer.render()` only called when `needsRender = true`; `controls.update()` runs every frame for damping inertia
- **Line2 from Three.js addons** — enables true pixel-width lines (WebGL core `linewidth` > 1 is broken on most platforms)
- **Procedural starfield** — Canvas 2D → `CanvasTexture` → inverted `SphereGeometry(500)`, zero external assets
- **Pointer-event drag** — `setPointerCapture` + `transform: translate()` only (no `top/left` repaints)
- **State lives in app.js only** — modules receive data via arguments and callbacks, never hold mutable shared state

## Notes for Next Version

Per user comment: future versions should **not auto-load** a data file. Instead, launch to a blank canvas and prompt the user to load/import a JSON data file. The current auto-load behaviour in `app.js → bootstrap()` is intentional for v0.1.x only.
