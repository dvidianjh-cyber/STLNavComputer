# Nav-Computer v0.2.0 ‚Äî Walkthrough

## Summary of Changes

All 7 files modified. Zero console errors. App verified at http://localhost:8080.

---

## Changes by File

### `data/systems.json`
- Wrapped the bare `[]` array in a root object with `map_name` and `map_description` metadata fields.

### `js/dataLoader.js`
- `loadSystems()` now returns `{ systems, mapName, mapDescription }` instead of a bare array.
- Added `MapData` typedef. Validation updated for object-based schema.

### `js/app.js`
- State expanded: `mapName`, `mapDescription`, `selectedSystem`, `hiddenLabelTypes` (Set), `currentRoute`.
- New callbacks wired: `onSystemSelect`, `onLabelToggle`, `onCenterView`.
- Calls `ui.setMapMeta()` after loading to push map name/description to header.
- Calls `renderer.setSelectedSystem()` on initial load and on dropdown change.

### `index.html`
- **New**: "Current System" panel (`#panel-current-system`) ‚Äî top-centre, with `<select>` and a ‚äï center-view button.
- **Modified**: Drive Output input replaced with `<input type="range">` + live value `<span>`.
- **Modified**: Drive Output and Max Cruise Velocity placed on one `param-dual-row` flex row (70%/30%).
- **Modified**: Journey Planner `aria-expanded` defaults to `"false"` (starts collapsed).
- **New**: Map Key panel with `#legend-items` container and a collapse button.

### `js/ui.js`
- `init()` extended with `onSystemSelect`, `onLabelToggle`, `onCenterView` callbacks and `initialSystem` param.
- `setMapMeta(mapName, mapDescription)` ‚Äî new export that replaces hardcoded header text.
- `_initCurrentSystemDropdown()` ‚Äî populates the dropdown and wires events.
- `_initMapKey()` ‚Äî dynamically derives unique system types, generates legend rows with coloured dots + labels + checkboxes.
- Journey Planner now starts collapsed on init.
- Legend collapse toggle wired (same pattern as planner).
- Drive Output slider live-display wired via `input` event.
- **Flash bug fix**: `pinCard()` now sets `opacity: 0` before appending, then uses `requestAnimationFrame` to set `opacity: 1` after positioning.
- **Removed** `card-id` `<span>` from pinned card header markup.

### `js/renderer.js`
- `loadStars()` now also builds CSS2DObject system name labels (class `system-label`), stored in `_labelObjects` Map.
- `_buildGrid()` now accepts `maxXZ` dataset extent and sizes the grid with 10% padding.
- `setSelectedSystem(system)` ‚Äî new export; moves the blue highlight sphere to system coordinates.
- `setHiddenLabelTypes(hiddenTypesSet)` ‚Äî new export; toggles `CSS2DObject.visible` for each label.
- `centerOnSystem(system)` ‚Äî new export; snaps `OrbitControls.target` to system coordinates.
- Renamed route CSS2D objects from `css2dObjects` to `css2dRouteObjects` to separate from star label Map.
- `'Unexplored': 0x888899` added to `STAR_COLORS`.

### `style.css`
- Version bumped to v0.2.0.
- `--col-unexplored` token added.
- `.panel-current-system`, `.current-system-select`, `.btn-center-view` styles.
- `.param-dual-row`, `.param-sub-group`, `.param-slider`, `.param-slider-value` styles.
- Legend collapse: `.panel-legend.is-collapsed`, `.legend-header`, `.legend-items`.
- `.legend-toggle-row`, `.legend-toggle-checkbox` for Map Key toggles.
- `.dot-*` classes expanded to match dynamically-generated CSS class names (e.g. `dot-habitable-hub`, `dot-unexplored`).
- `tooltip-type-unexplored` and `type-unexplored` badge styles.
- `pinned-card` animation replaced with `transition: opacity 0.12s ease` (flash bug fix).
- `.system-label` CSS2D label styles.
