# Nav-Computer v0.2.0 Implementation Plan

This update expands the data schema, adds new UI controls, introduces 3D system name labels, dynamic map scaling, fixes the pinned card flash bug, and centralises state management. All changes preserve the zero-build-step architecture (ES6 modules, no bundler).

---

## User Review Required

> [!IMPORTANT]
> **Journey Planner — Drive Output Slider**: The spec says max = 10 G (slider). The current `param-range` label and validation clamps at 0.1–100 G. The slider will visually limit to 10 G, but the `_handleCalculate` logic will use the raw slider value (0.1–10). Confirm this is acceptable, or should the existing number input remain alongside the slider for values > 10 G?

> [!WARNING]
> **`systems.json` schema change is a breaking change** — the file is going from a bare `[]` array to a `{ "map_name": ..., "systems": [] }` object. Once updated, any external tooling or scripts relying on the raw array format will break. Ensure this is intentional.

> [!NOTE]
> **CSS2DObject system labels** — the spec prefers `CSS2DRenderer` for text clarity. The renderer already imports and uses `CSS2DRenderer` for route leg-time labels, so this adds zero new dependencies.

### 4 · Renderer Changes

#### [MODIFY] [renderer.js](file:///c:/Users/dvidi/Documents/Progamming/STLNavComputer/js/renderer.js)
- **`loadStars(systems)`**: After building star meshes, also call `_buildSystemLabels(systems)` to create one `CSS2DObject` per star with class `system-label`. Store them in a module-level map `_labelObjects = new Map<systemId, CSS2DObject>`.
- **`_buildGrid(systems)`** → make `_buildGrid()` accept systems data: calculate `maxDist = max of |x| and |z| across all systems`, set grid size to `Math.ceil(maxDist * 1.1)`. Update the call in `initScene()` to defer until `loadStars()` is called (or pass systems to `loadStars` which already has them).
- **`setSelectedSystem(system)`** (new export): Moves the blue highlight sphere to `system.coordinates`. The sphere is initialised at origin (current v0.1 behaviour); this function repositions it. Add a module-level `_highlightSphere` created in `initScene()`.
- **`setHiddenLabelTypes(hiddenTypesSet)`** (new export): Iterates `_labelObjects`, sets each `CSS2DObject.visible = !hiddenTypesSet.has(system.type)`. Calls `markDirty()`.
- **Unexplored star color**: Add `'Unexplored': 0x888899` to `STAR_COLORS`.
