# STL Nav Computer — v0.1.2 Walkthrough\n\n## Changes Implemented\n\n### 1. Bug Fix — Close Button on Pinned Cards\n**Root cause:** The header's `pointerdown` drag handler was capturing pointer events before the close button's `click` could fire.\n**Fix (ui.js):**\n- `e.stopPropagation()` on close button's `pointerdown`.\n- Early-return guard in `_makeDraggable` pointerdown: `if (e.target.closest('.card-close')) return;`\n- `_destroyTether(system.id)` wired inside the close handler.\n\n---\n\n### 2a. Collapsible Journey Planner\n- `<button id=\"btn-collapse-planner\">⌄</button>` added inside the panel heading.\n- Click toggles `.is-collapsed` on `#panel-planner`; body animates away via `max-height` transition.\n- Icon rotates 180° when collapsed via CSS.\n\n---\n\n### 2b. Per-Waypoint Layover (years, step 0.1)\n- Global `#layover-input` HTML block and `$layover` accessor removed.\n- `_stops` schema extended: `{ stopId, systemId, layoverYears: 0 }`.\n- Each intermediate stop gets a `.stop-layover-row` with number input (step=0.1, min=0) and \"YRS\" unit.\n- `_handleCalculate` builds `layoverYearsArray` in parallel with `waypointSystems`.\n- `calculateRoute(waypoints, gForce, maxCruise, layoverYearsArray)` — no more day→year conversion, values are used directly.\n\n---\n\n### 3. Theme Contrast\n| Token | Before | After |\n|---|---|---|\n| `--col-bg` | `#000a1a` | `#00060f` |\n| `--col-white` | `#e8f4ff` | `#f5faff` |\n| `--col-muted` | `rgba(180,210,255,0.45)` | `rgba(190,218,255,0.62)` |\n\nHard-coded hex in `renderer.js` (fog, clear color, starfield fill) updated to `#00060f`.\n\n---\n\n### 4. Tether Lines (Star → Pinned Card, Closest-Edge)\n- Full-screen `<svg id=\"tether-layer\">` at z-index 5, `pointer-events: none`.\n- Dashed cyan SVG `<line class=\"tether-line\">` per pinned card.\n- **Closest-edge anchor:** clamps star screen-pos to card bounding rect:\n  `cx = clamp(starX, cardX, cardX+cardW)` / `cy = clamp(starY, cardY, cardY+cardH)`.\n- Updates on card drag, orbit controls change, and window resize.\n- `renderer.worldToScreen(coords)` projects 3D star coords to screen pixels.\n- `renderer.getControls()` exposes OrbitControls for `change` event subscription in `app.js`.\n\n---\n\n### 5. Flight Plan Leg Labels (CSS2DRenderer)\n- `CSS2DRenderer` overlaid as `position:fixed; z-index:5; pointer-events:none`.\n- Each leg midpoint gets a `CSS2DObject` wrapping a `.leg-label` pill div showing `formatTime(leg.tSubj)`.\n- Zero-distance legs are skipped.\n- Labels cleared and recreated on every route calculation; `css2dRenderer.render(scene, camera)` called each dirty frame.\n\n---\n\n## Files Changed\n| File | Changes |\n|---|---|\n| js/ui.js | Bug fix, collapse, layover schema, tether system, `refreshTethers` export |\n| js/physics.js | `calculateRoute` → `layoverYearsArray` |\n| js/renderer.js | CSS2DRenderer, leg labels, `worldToScreen`, `getControls`, theme colors |\n| js/app.js | `layoverYearsArray` forwarding, tether subscription wiring |\n| style.css | All token + new component styles |\n| index.html | Theme-color, SVG layer, collapse button, remove layover block |\n\n## Verification Checklist\n- [ ] Click × on pinned card → closes cleanly, tether disappears\n- [ ] Collapse toggle → planner animates, icon flips\n- [ ] Add intermediate stops → each has a LAYOVER YRS input affecting total\n- [ ] Text brighter, background darker\n- [ ] Pin star → dashed tether line to card; drag card → line follows; pan map → line re-anchors\n- [ ] Multi-waypoint route → subjective time label at each leg midpoint; clear → gone\n
